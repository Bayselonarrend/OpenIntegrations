# Upload file
 Upload single file to Neocities



<Tabs>
  <TabItem value="params" label="Parameters" default>

`Function UploadFile(Val Token, Val Path, Val Data) Export`

  | Parameter | CLI option | Type | Required | Description |
  |-|-|-|-|-|
  | Token | --token | String | &#x2714; | Auth token |
  | Path | --path | String | &#x2714; | File path on Neocities |
  | Data | --file | String, BinaryData | &#x2714; | URL, path or file data |


<div className="return-value-note">
  <div className="return-value-note__title">Returns</div>
  <div className="return-value-note__value">
 Map Of KeyAndValue - serialized JSON response from Neocities
  </div>
</div>
  
  </TabItem>
  <TabItem value="extended" label={<span>Advanced call{' '}<a href="/docs/Start/Advanced-call" target="_blank" rel="noreferrer" title="About advanced call" onClick={(e) => e.stopPropagation()}>?</a></span>}>

  | Parameter | Description |
  |---|---|
  | dontwait | Creates a background job and returns its object (for 1C and OneScript only) |


  </TabItem>
</Tabs>

:::tip
Neocities API docs: [neocities.org/api](https://neocities.org/api)

Parameters with Binary data type can also accept file paths on disk and URLs
:::
<br/>


```bsl title="1C:Enterprise/OneScript code example"
    Token = "f1e616e20ab5893b...";
    Data  = "https://hut.openintegrations.dev/test_data/picture.jpg"; // URL, Path or Binary data
    Path  = "testfolder/test_pic.png";

    Result = OPI_Neocities.UploadFile(Token, Path, Data);
```
