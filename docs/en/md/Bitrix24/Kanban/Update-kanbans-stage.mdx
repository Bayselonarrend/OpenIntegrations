# Change kanban stage
 Changes the properties of the existing kanban or My plan stage



<Tabs>
  <TabItem value="params" label="Parameters" default>

`Function UpdateKanbansStage(Val URL, Val Name, Val StageID, Val Color = "", Val PrevStageID = 0, Val AsAdmin = False, Val Token = "") Export`

  | Parameter | CLI option | Type | Required | Description |
  |-|-|-|-|-|
  | URL | --url | String | &#x2714; | URL of webhook or a Bitrix24 domain, when token used |
  | Name | --title | String | &#x2714; | New stages name |
  | StageID | --stage | String, Number | &#x2714; | Stage ID for change |
  | Color | --color | String | &#x2716; | HEX of new stage color |
  | PrevStageID | --prevstage | String, Number | &#x2716; | Stage ID, after which the selected stage should be inserted |
  | AsAdmin | --admin | Boolean | &#x2716; | Allows you to add stages without checking permissions (for administrators) |
  | Token | --token | String | &#x2716; | Access token, when app auth method used |
