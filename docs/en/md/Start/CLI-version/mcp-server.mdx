# Built-in MCP server

The **MCP server** in OInt CLI is a built-in implementation of the [Model Context Protocol](https://modelcontextprotocol.io/) over standard input/output (stdio) that you can use to let an AI agent work directly with the capabilities of the Open Integration Package.

## Available tools

| Tool | Purpose |
|------|---------|
| `get_services` | Lists available services (CLI commands) matching OInt libraries |
| `get_service_methods` | Lists methods for the selected service |
| `get_method_parameters` | Describes method parameters (types, default values, hints) |
| `execute_method` | Runs a method with a set of named parameters; the result is a string (as in CLI) |
| `get_variables` | Returns the list of variables defined in the configuration file |

For `execute_method`, parameter values are strings. If a value starts with `@@`, it is treated as a key for a variable from the configuration `vars` block (see below), which is convenient for using secrets without passing them directly to the agent.

## Starting the server

Start the MCP server with `oint mcp start` and one or more parameters:

- `--config` (`-c`) — path to a JSON configuration file, or the literal `default` for the default configuration
- `--log` (`-l`) — path to a log file, if required

Example start commands:

<Tabs>
  <TabItem value="bash" label="Bash" default>
```bash
oint mcp start --config default
oint mcp start -c /home/user/oint-mcp.json -l /tmp/oint-mcp.log
```
  </TabItem>
  <TabItem value="bat" label="CMD/Bat">
```batch
oint mcp start --config default
```
```batch
oint mcp start -c C:\Users\user\oint-mcp.json -l C:\Temp\oint-mcp.log
```
  </TabItem>
  <TabItem value="powershell" label="PowerShell">
```powershell
oint mcp start --config default
```
```powershell
oint mcp start -c "$env:USERPROFILE\oint-mcp.json" -l "$env:TEMP\oint-mcp.log"
```

</Tabs>

You can obtain a template for your own configuration (the `--config` argument) with the separate command `oint mcp config`:

<Tabs>
  <TabItem value="bash" label="Bash" default>
```bash
oint mcp config --empty true
```
  </TabItem>
  <TabItem value="bat" label="CMD/Bat">
```batch
oint mcp config --empty true
```
  </TabItem>
  <TabItem value="powershell" label="PowerShell">
```powershell
oint mcp config --empty true
```

  </TabItem>

With `--empty true` (the default) a minimal skeleton is returned; with `--empty false`, explanatory example fields are added under `vars`. Save the template using the standard `--out <file path>` option or shell redirection.

## Configuration file

The configuration is a JSON file for MCP server settings. Currently a single block is supported — **`vars`**, which holds user-defined variable values. The agent can use these at runtime and substitute them for real data when calling the `execute_method` tool.

Example `oint-mcp.json`:

```json title="oint-mcp.json"
{
  "vars": {
    "TelegramToken": "123456:ABC-DEF...",
    "BitrixWebhook": "https://example.bitrix24.ru/rest/1/xxxxx/"
  }
}
```

Starting the server with a configuration file:

<Tabs>
  <TabItem value="bash" label="Bash" default>
```bash
oint mcp start -c /home/user/oint-mcp.json
```
  </TabItem>
  <TabItem value="bat" label="CMD/Bat">
```batch
oint mcp start -c C:\Users\user\oint-mcp.json
```
  </TabItem>
  <TabItem value="powershell" label="PowerShell">
```powershell
oint mcp start -c "$env:USERPROFILE\oint-mcp.json"
```
  </TabItem>
</Tabs>

While running, the agent can call the `get_variables` tool to obtain the names of defined variables (not their actual values). Each name is prefixed with `@@` and can be supplied as a value for OInt function parameters. When the MCP server receives a value starting with `@@`, it treats it as a variable key and replaces it with the value from the configuration file if that key exists. Secrets are therefore not passed directly to the AI agent.

:::caution
Do not publish files with real tokens in repositories!
:::

## Connecting the server in AI agents

Below is a typical **local** MCP server snippet: the `oint` command with arguments `mcp`, `start`, and configuration parameters. Replace `oint` with the full path to the executable if needed (especially on Windows when `PATH` is not set).

<Tabs>
  <TabItem value="cursor" label="Cursor" default>
In Cursor, MCP is configured in user or project settings (for example via the **MCP** settings in the UI or a file under `.cursor`).

```json
{
  "mcpServers": {
    "oint-mcp": {
      "command": "oint",
      "args": ["mcp", "start", "-c", "default"]
    }
  }
}
```

For a configuration file that holds secrets, use an absolute path instead of `default`:

```json
{
  "mcpServers": {
    "oint-mcp": {
      "command": "oint",
      "args": ["mcp", "start", "-c", "C:\\Users\\user\\oint-mcp.json", "-l", "C:\\Temp\\oint-mcp.log"]
    }
  }
}
```

See the [Cursor MCP documentation](https://docs.cursor.com/context/mcp) for the current format and file location.
  </TabItem>
  <TabItem value="kiro" label="Kiro">
Kiro uses:

- project-level: `.kiro/settings/mcp.json`
- user-level: `~/.kiro/settings/mcp.json`

When both exist, they are merged; workspace settings take precedence. Example local server:

```json
{
  "mcpServers": {
    "oint-mcp": {
      "command": "oint",
      "args": ["mcp", "start", "-c", "default"]
    }
  }
}
```

For a configuration file with secrets, use an absolute path instead of `default`:

```json
{
  "mcpServers": {
    "oint-mcp": {
      "command": "oint",
      "args": ["mcp", "start", "-c", "C:\\Users\\user\\oint-mcp.json", "-l", "C:\\Temp\\oint-mcp.log"]
    }
  }
}
```

Create or edit the config via the command palette (**Kiro: Open workspace MCP config** / **Kiro: Open user MCP config**). Details: [Configuration](https://kiro.dev/docs/mcp/configuration/) in the Kiro documentation.
  </TabItem>
  <TabItem value="cline" label="Cline">
In Cline, servers are defined in JSON under the `mcpServers` key. File locations may change between extension versions; follow the current guide in the [Cline documentation](https://docs.cline.bot/mcp/configuring-mcp-servers). Example for a stdio server:

```json
{
  "mcpServers": {
    "oint-mcp": {
      "command": "oint",
      "args": ["mcp", "start", "-c", "default"],
      "disabled": false
    }
  }
}
```

For a configuration file with secrets, use an absolute path instead of `default`:

```json
{
  "mcpServers": {
    "oint-mcp": {
      "command": "oint",
      "args": ["mcp", "start", "-c", "C:\\Users\\user\\oint-mcp.json", "-l", "C:\\Temp\\oint-mcp.log"]
    }
  }
}
```

Server management (enable, restart) is available from the MCP panel in the Cline UI.
  </TabItem>
</Tabs>

## Troubleshooting

If the client does not see the tools or fails on startup, check in a normal terminal:

1. Whether `oint mcp start -c default` runs without errors (the process should wait for input and not print messages to the console).
2. That the JSON in the configuration file is valid when using `-c` with a path.
3. The log when `-l` is set: it records request processing steps and error text.
