# Изменить пользователя
 Изменяет данные существуюего пользователя



`Функция ИзменитьПользователя(Знач Соединение, Знач Имя, Знач РолиПользователя = Неопределено, Знач База = Неопределено, Знач Пароль = Неопределено, Знач Параметры = Неопределено) Экспорт`

  | Параметр | CLI опция | Тип | Обяз. | Назначение |
  |-|-|-|-|-|
  | Соединение | --dbc | Строка, Произвольный | &#x2714; | Соединение или строка подключения |
  | Имя | --name | Строка | &#x2714; | Имя пользователя |
  | РолиПользователя | --roles | Массив Из Произвольный | &#x2716; | Массив ролей в виде строк или структур с указанием БД |
  | База | --db | Строка | &#x2716; | Имя базы. Текущая база, если не указано |
  | Пароль | --pwd | Строка | &#x2716; | Пароль пользователя |
  | Параметры | --params | Структура Из КлючИЗначение | &#x2716; | Дополнительные параметры |

  
  Возвращаемое значение:   Соответствие Из КлючИЗначение - Результат выполнения операции

:::tip
Переданные роли целиком заменяют список ролей пользователя

Роль может быть определена в одном из следующих форматов:<br/>Структура: `{ role: "имя", db: "база данных" }`<br/>Строка:    `имя роли`
:::
<br/>


```bsl title="Пример использования для 1С:Предприятие/OneScript"
    Адрес  = "127.0.0.1:1234";
    Логин  = "bayselonarrend";
    Пароль = "12we...";
    База   = "main";

    ПараметрыПодключения = Новый Структура("authSource", "admin");
    СтрокаПодключения    = OPI_MongoDB.СформироватьСтрокуПодключения(Адрес, , Логин, Пароль, ПараметрыПодключения);
    Соединение           = OPI_MongoDB.ОткрытьСоединение(СтрокаПодключения);

    МассивРолей = Новый Массив;
    МассивРолей.Добавить("readWrite");

    ИмяПользователя    = "newuser";
    ПарольПользователя = "4321";

    Результат = OPI_MongoDB.ИзменитьПользователя(Соединение, ИмяПользователя, МассивРолей, База, ПарольПользователя);
```
