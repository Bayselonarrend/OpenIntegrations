# Архивировать каталог
 Запаковывает файлы указанного каталога в tar-архив



<Tabs>
  <TabItem value="params" label="Параметры" default>

`Функция АрхивироватьКаталог(Знач Каталог, Знач ПутьАрхива = "", Знач Настройки = Неопределено) Экспорт`

  | Параметр | CLI опция | Тип | Обяз. | Назначение |
  |-|-|-|-|-|
  | Каталог | --src | Строка, ДвоичныеДанные | &#x2714; | Путь к каталогу или описание данных для запаковки |
  | ПутьАрхива | --dest | Строка | &#x2716; | Путь сохранения архива. В двоичные данные, если не заполнено |
  | Настройки | --settings | Соответствие Из КлючИЗначение | &#x2716; | Доп. настройки. См. [`ПолучитьСтруктуруНастроекАрхивации`](/docs/Tar/Archiving/Get-archiving-settings-structure) |


<div className="return-value-note">
  <div className="return-value-note__title">Возвращаемое значение</div>
  <div className="return-value-note__value">
  ДвоичныеДанные, Соответствие Из КлючИЗначение - Информация о выполнении или двоичные данные архива
  </div>
</div>
  
  </TabItem>
  <TabItem value="extended" label={<span>Расширенный вызов{' '}<a href="/docs/Start/Advanced-call" target="_blank" rel="noreferrer" title="Справка о расширенном вызове" onClick={(e) => e.stopPropagation()}>?</a></span>}>

  | Параметр | Описание |
  |---|---|
  | dontwait | Создает фоновое задание и возвращает его данные (только для 1С и OneScript) |


  </TabItem>
</Tabs>

:::tip
Параметры с типом данных Двоичные данные могут принимать также пути к файлам на диске и URL
:::
