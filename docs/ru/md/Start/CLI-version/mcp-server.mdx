# Встроенный MCP-сервер

**MCP-сервер** в OInt CLI — это встроенная реализация [Model Context Protocol](https://modelcontextprotocol.io/) поверх стандартных потоков ввода-вывода (stdio), которую можно использовать для организации прямой работы ИИ-агента с различными возможностями Открытого пакета интеграций. 

## Доступные инструменты

| Инструмент | Назначение |
|------------|------------|
| `get_services` | Список доступных сервисов (команд CLI), соответствующих библиотекам OInt |
| `get_service_methods` | Список методов выбранного сервиса |
| `get_method_parameters` | Описание параметров метода (типы, значения по умолчанию, подсказки) |
| `execute_method` | Выполнение метода с набором именованных параметров; результат — строка (как в CLI) |
| `get_variables` | Получение списка переменных, установленных в файле конфигурации |

При вызове `execute_method` значения параметров — строки. Если значение начинается с `@@`, оно трактуется как ключ переменной из блока `vars` конфигурации (см. ниже), что удобно для использования различных секретных данных без их прямой передачи агенту

## Запуск сервера

MCP сервер запускается при помощи команды `oint mcp start` с одним или несколькими параметрами

- `--config` (`-c`) — путь к JSON-файлу конфигурации или строка `default` для стандартной конфигурации
- `--log` (`-l`) — путь к файлу для сохранения лога, если необходимо.

Примеры команд запуска:

<Tabs>
  <TabItem value="bash" label="Bash" default>
```bash
oint mcp start --config default
oint mcp start -c /home/user/oint-mcp.json -l /tmp/oint-mcp.log
```
  </TabItem>
  <TabItem value="bat" label="CMD/Bat">
```batch
oint mcp start --config default
```
```batch
oint mcp start -c C:\Users\user\oint-mcp.json -l C:\Temp\oint-mcp.log
```
  </TabItem>
  <TabItem value="powershell" label="PowerShell">
```powershell
oint mcp start --config default
```
```powershell
oint mcp start -c "$env:USERPROFILE\oint-mcp.json" -l "$env:TEMP\oint-mcp.log"
```
  </TabItem>
</Tabs>

Шаблон для создания своей конфигурации (параметр `--config`) можно получить отдельной командой `oint mcp config`:

<Tabs>
  <TabItem value="bash" label="Bash" default>
```bash
oint mcp config --empty true
```
  </TabItem>
  <TabItem value="bat" label="CMD/Bat">
```batch
oint mcp config --empty true
```
  </TabItem>
  <TabItem value="powershell" label="PowerShell">
```powershell
oint mcp config --empty true
```
  </TabItem>
</Tabs>

При `--empty true` (значение по умолчанию) возвращается минимальный каркас; при `--empty false` в `vars` добавляются поясняющие примеры полей. Сохранить файл шаблона можно при помощи добавления стандартного параметра `--out <путь к файлу>` или используя перенаправление оболочки

## Файл конфигурации

Конфигурация - это JSON файл, предназначенный для хранения различных настроек MCP-сервера. На данный момент там реализован один блок - `vars`, который предназначен для хранения значений пользовательских переменных. Эти переменные могут быть использованы агентом в процессе работы для их подстановки вместо настоящих данных при вызове инструмента `execute_method`

Пример файла `oint-mcp.json`:

```json title="oint-mcp.json"
{
  "vars": {
    "TelegramToken": "123456:ABC-DEF...",
    "BitrixWebhook": "https://example.bitrix24.ru/rest/1/xxxxx/"
  }
}
```

Запуск сервера с указанием файла конфигурации:

<Tabs>
  <TabItem value="bash" label="Bash" default>
```bash
oint mcp start -c /home/user/oint-mcp.json
```
  </TabItem>
  <TabItem value="bat" label="CMD/Bat">
```batch
oint mcp start -c C:\Users\user\oint-mcp.json
```
  </TabItem>
  <TabItem value="powershell" label="PowerShell">
```powershell
oint mcp start -c "$env:USERPROFILE\oint-mcp.json"
```
  </TabItem>
</Tabs>

В процессе работы агент может вызвать инструмент `get_variables` для получения списка названий установленных переменных (без реальных значений). Все они будут начинаться с символов `@@` и могут быть установлены им в качестве значений переменных функций OInt. А MCP сервер при получении значения, начинающегося на `@@`, интерпретирует его как ключ переменной и заменяет на реальное значение из файла конфигурации, если переменная с таким ключом существует. Таким образом секретные данные не передаются напрямую ИИ-агенту

:::caution
Не публикуйте файлы с реальными токенами в репозиториях!
:::

## Подключение сервера в ИИ-агентах

Ниже представлен типовой фрагмент **локального** MCP-сервера: команда `oint` и аргументы `mcp`, `start`, параметры конфигурации. При необходимости замените `oint` на полный путь к исполняемому файлу (особенно на Windows, если `PATH` не настроен).

<Tabs>
  <TabItem value="cursor" label="Cursor" default>
В Cursor MCP задается в пользовательском или проектном конфиге (например, через настройки **MCP** в интерфейсе или файл в каталоге `.cursor`)

```json
{
  "mcpServers": {
    "oint-mcp": {
      "command": "oint",
      "args": ["mcp", "start", "-c", "default"]
    }
  }
}
```

Для файла конфигурации с секретами укажите абсолютный путь вместо `default`:

```json
{
  "mcpServers": {
    "oint-mcp": {
      "command": "oint",
      "args": ["mcp", "start", "-c", "C:\\Users\\user\\oint-mcp.json", "-l", "C:\\Temp\\oint-mcp.log"]
    }
  }
}
```

Актуальный формат и расположение файла см. в [документации Cursor по MCP](https://docs.cursor.com/context/mcp).
  </TabItem>
  <TabItem value="kiro" label="Kiro">
В Kiro используются файлы:

- уровня проекта: `.kiro/settings/mcp.json`
- уровня пользователя: `~/.kiro/settings/mcp.json`

При наличии обоих они объединяются, приоритет отдается настройкам рабочей области. Пример локального сервера:

```json
{
  "mcpServers": {
    "oint-mcp": {
      "command": "oint",
      "args": ["mcp", "start", "-c", "default"]
    }
  }
}
```

Для файла конфигурации с секретами укажите абсолютный путь вместо `default`:

```json
{
  "mcpServers": {
    "oint-mcp": {
      "command": "oint",
      "args": ["mcp", "start", "-c", "C:\\Users\\user\\oint-mcp.json", "-l", "C:\\Temp\\oint-mcp.log"]
    }
  }
}
```

Создать или отредактировать конфиг можно через палитру команд (**Kiro: Open workspace MCP config** / **Kiro: Open user MCP config**). Подробности: [Configuration](https://kiro.dev/docs/mcp/configuration/) в документации Kiro.
  </TabItem>
  <TabItem value="cline" label="Cline">
В Cline список серверов задается в JSON с ключом `mcpServers`. Расположение файлов может обновляться между версиями расширения; ориентируйтесь на актуальную инструкцию в [документации Cline](https://docs.cline.bot/mcp/configuring-mcp-servers). Пример для stdio-сервера:

```json
{
  "mcpServers": {
    "oint-mcp": {
      "command": "oint",
      "args": ["mcp", "start", "-c", "default"],
      "disabled": false
    }
  }
}
```

Для файла конфигурации с секретами укажите абсолютный путь вместо `default`:

```json
{
  "mcpServers": {
    "oint-mcp": {
      "command": "oint",
      "args": ["mcp", "start", "-c", "C:\\Users\\user\\oint-mcp.json", "-l", "C:\\Temp\\oint-mcp.log"]
    }
  }
}
```

Управление серверами (включение, перезапуск) доступно из панели MCP в интерфейсе Cline.
  </TabItem>
</Tabs>

## Диагностика

Если клиент не видит инструменты или падает при старте, проверьте в обычном терминале:

1. Выполняется ли `oint mcp start -c default` без ошибок (процесс должен ожидать ввод, не выводя сообщения в консоль).
2. Корректность JSON в файле конфигурации при использовании `-c` с путем.
3. Журнал при указании `-l`: в нем фиксируются этапы обработки запросов и текст ошибок.
