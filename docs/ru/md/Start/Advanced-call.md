# Расширенный вызов

Некоторые методы ОПИ обращаются к сети (HTTP-клиент и аналогичные сценарии). Для них доступен **расширенный вызов** — передача дополнительных настроек, которые не входят в обычную сигнатуру функции: прокси, таймаут, формат «расширенного» ответа и другие параметры. Они перечислены на **второй вкладке** страницы описания метода (вкладка «Расширенный вызов»).

Состав доступных настроек **зависит от конкретного метода**: у части функций расширенных параметров нет — это отражено в документации на странице метода.

## 1С:Предприятие и OneScript

Программный интерфейс — модуль `OPI_РасширенныйВызов`:

- **`ВызватьСНастройками`** — вызывает метод модуля ОПИ с передачей структуры настроек (и массива аргументов метода в исходном порядке).
- **`ПолучитьДоступныеНастройки`** — возвращает текстовое описание доступных для пары «модуль + метод» ключей настроек.

Настройки задаются коллекцией «ключ — значение»; имена ключей и смысл значений совпадают с описанием на вкладке «Расширенный вызов» в документации соответствующего метода (например, `proxy`, `timeout`, `adv_response` там, где они поддерживаются).

## CLI

Стандартная опция **`--config`** задаёт дополнительные параметры вызова метода. Если вызвать метод с **`--config` без значения**, приложение выведет список доступных для этого метода настроек (тот же набор, что и в документации на вкладке «Расширенный вызов»).

Подробнее об общих опциях CLI см. раздел [Работа с CLI версией](/docs/Start/CLI-version).

## Где смотреть в документации

На странице каждого метода блок параметров разбит на две вкладки:

1. **Параметры** — объявление функции и таблица обычных параметров (и CLI-опций).
2. **Расширенный вызов** — таблица дополнительных настроек либо сообщение о том, что для метода таких настроек нет.

Ссылка «**?**» рядом с названием второй вкладки ведёт на эту страницу.
