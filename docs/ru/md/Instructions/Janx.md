# Janx

import LibraryIntro from '@site/src/components/LibraryIntro';

<LibraryIntro module="OPI_Janx" cli="janx" use="oint/formats/janx" lang="ru"/>

Этот раздел посвящен библиотеке для работы с форматом Janx в 1С:Предприятие, OneScript и CLI. На данной странице описаны основные возможности и примеры использования

## Описание формата

Janx-1 — это бинарный формат сериализации данных, оптимизированный для эффективного хранения и передачи структурированных данных с поддержкой двоичных данных.

### Структура формата

Формат Janx-1 имеет следующую структуру:
- **Версия** (u16 BE) — версия формата
- **Длина JSON** (u32 BE) — размер JSON части в байтах
- **Число маркеров** (u32 BE) — количество маркеров двоичных данных
- **JSON** — структурированные данные в формате JSON
- **Приложение** — двоичные данные, на которые ссылаются маркеры в JSON

### Особенности формата

- **Поддержка двоичных данных** — двоичные данные хранятся отдельно от JSON и замещаются маркерами
- **Эффективное хранение** — двоичные данные не кодируются в Base64, что экономит место
- **Поддержка коллекций** — работает со структурами, соответствиями, массивами и вложенными коллекциями
- **Порядок байтов Big Endian** — обеспечивает кроссплатформенность

## Использование

### Сериализация данных

Для преобразования коллекции данных в формат Janx используйте функцию `СериализоватьДанные`:

```bsl
Данные = Новый Структура;
Данные.Вставить("Название", "Тестовый документ");
Данные.Вставить("Номер", 12345);
Данные.Вставить("Файл", ПолучитьДвоичныеДанныеИзФайла("документ.pdf"));

СериализованныеДанные = OPI_Janx.СериализоватьДанные(Данные);
// СериализованныеДанные — двоичные данные в формате Janx-1
```

Функция принимает любую коллекцию (Структура, Соответствие, Массив) и возвращает двоичные данные в формате Janx-1.

### Десериализация данных

Для восстановления данных из формата Janx используйте функцию `ДесериализоватьДанные`:

```bsl
ВосстановленныеДанные = OPI_Janx.ДесериализоватьДанные(СериализованныеДанные);
// ВосстановленныеДанные — исходная коллекция с восстановленными двоичными данными
```

Функция принимает двоичные данные в формате Janx-1 и возвращает десериализованную коллекцию.

### Работа с вложенными структурами

Формат Janx поддерживает сериализацию сложных вложенных структур:

```bsl
Документ = Новый Структура;
Документ.Вставить("Заголовок", "Отчет");
Документ.Вставить("Дата", ТекущаяДата());

Вложения = Новый Массив;
Вложения.Добавить(ПолучитьДвоичныеДанныеИзФайла("файл1.pdf"));
Вложения.Добавить(ПолучитьДвоичныеДанныеИзФайла("файл2.jpg"));

Документ.Вставить("Вложения", Вложения);

СериализованныеДанные = OPI_Janx.СериализоватьДанные(Документ);
ВосстановленныеДанные = OPI_Janx.ДесериализоватьДанные(СериализованныеДанные);
```

### Поддерживаемые типы данных

Формат Janx поддерживает следующие типы данных:

- **Примитивные типы**: Строка, Число, Булево, Дата, Неопределено
- **Коллекции**: Структура, Соответствие, Массив
- **Двоичные данные**: ДвоичныеДанные

При сериализации:
- Примитивные типы сохраняются в JSON части
- Коллекции обрабатываются рекурсивно
- ДвоичныеДанные выносятся в приложение и замещаются маркерами

## Преимущества использования

1. **Компактность** — двоичные данные не кодируются в Base64, что экономит ~33% места
2. **Производительность** — быстрая сериализация и десериализация
3. **Структурированность** — JSON часть легко читается и отлаживается
4. **Универсальность** — работает с любыми вложенными коллекциями
5. **Типизация** — сохраняет типы данных при сериализации/десериализации

## Области применения

- Обмен данными между системами
- Сохранение сложных структур в базах данных
- Кэширование данных с двоичными компонентами
- Передача данных через HTTP/REST API
- Архивирование структурированных данных
