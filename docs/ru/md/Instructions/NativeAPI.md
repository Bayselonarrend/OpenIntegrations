## Начало работы

Библиотека предоставляет возможность работы с нативными библиотеками (DLL/SO) через специальную компоненту-прослойку. Это позволяет вызывать функции и работать с объектами внешних компонент без необходимости создания обёрток.

### Основные возможности

**Создание хоста и загрузка библиотеки**

Функция `СоздатьХост` создаёт объект прослойки Native API и загружает указанную DLL/SO библиотеку. При необходимости можно настроить логирование работы компоненты.

```bsl
ПутьКБиблиотеке = "C:\MyLib\component.dll";

// Без логирования
Хост = OPI_NativeAPI.СоздатьХост(ПутьКБиблиотеке);

// С настройкой логирования
НастройкиЛога = OPI_NativeAPI.ПолучитьНастройкиЛогирования(Истина, 500, "C:\Logs\nativeapi.log");
Хост = OPI_NativeAPI.СоздатьХост(ПутьКБиблиотеке, НастройкиЛога);
```

**Получение информации о классах**

Функция `ПолучитьИменаКлассов` возвращает список классов, объявленных в загруженной компоненте.

```bsl
СписокКлассов = OPI_NativeAPI.ПолучитьИменаКлассов(Хост);
```

### Работа с экземплярами компоненты

**Создание и уничтожение экземпляров**

Функция `СоздатьЭкземпляр` создаёт объект указанного класса компоненты и сохраняет его в хосте. Функция возвращает UUID экземпляра, который используется для дальнейшей работы с ним.

```bsl
ИмяКласса = "MyComponent.MainClass";
Экземпляр = OPI_NativeAPI.СоздатьЭкземпляр(Хост, ИмяКласса);

// UUID экземпляра в Экземпляр["data"]
УникальныйИдентификатор = Экземпляр["data"];

// После завершения работы
OPI_NativeAPI.УничтожитьЭкземпляр(Хост, УникальныйИдентификатор);
```

**Получение метаданных**

Функция `ПолучитьМетаданные` возвращает описание методов и свойств экземпляра компоненты или класса.

```bsl
// Для существующего экземпляра
Метаданные = OPI_NativeAPI.ПолучитьМетаданные(Хост, УникальныйИдентификатор);

// Для класса (без создания экземпляра)
Метаданные = OPI_NativeAPI.ПолучитьМетаданные(Хост, "MyComponent.MainClass");
```

### Вызов методов и работа со свойствами

**Вызов методов**

Функция `ВызватьМетод` позволяет вызывать методы экземпляра компоненты. Можно передавать параметры и устанавливать значения свойств перед вызовом.

```bsl
ИмяМетода = "CalculateSum";
Параметры = Новый Массив;
Параметры.Добавить(10);
Параметры.Добавить(20);

// Установка свойств перед вызовом (опционально)
Свойства = Новый Соответствие;
Свойства.Вставить("Precision", 2);

Результат = OPI_NativeAPI.ВызватьМетод(Хост, УникальныйИдентификатор, ИмяМетода, Параметры, Свойства);

// Результат содержит возвращаемое значение и текущие значения всех свойств
ЗначениеМетода = Результат["data"]["return"];
ЗначениеСвойства = Результат["data"]["properties"]["SomeProperty"];
```

**Работа со свойствами**

Библиотека предоставляет функции для чтения и записи свойств экземпляра:

```bsl
// Получение всех свойств
ВсеСвойства = OPI_NativeAPI.ПолучитьСвойства(Хост, УникальныйИдентификатор);

// Получение конкретного свойства
Значение = OPI_NativeAPI.ПолучитьСвойство(Хост, УникальныйИдентификатор, "PropertyName");

// Установка свойства
НовоеЗначение = 42;
OPI_NativeAPI.УстановитьСвойство(Хост, УникальныйИдентификатор, "PropertyName", НовоеЗначение);
```

### Временные экземпляры

Для одноразовых вызовов можно использовать временные экземпляры, передавая имя класса вместо UUID:

```bsl
ИмяКласса = "MyComponent.UtilityClass";
ИмяМетода = "GetVersion";

// Экземпляр будет создан, метод вызван и экземпляр уничтожен автоматически
Результат = OPI_NativeAPI.ВызватьМетод(Хост, ИмяКласса, ИмяМетода);
```

### Логирование

Функция `ПолучитьНастройкиЛогирования` возвращает структуру с настройками логирования компоненты:

- **ЗаписьВПамять** — записывать лог в память компоненты (по умолчанию `Истина`)
- **МаксимумСобытий** — максимальное количество событий в памяти (по умолчанию 300)
- **ПутьКФайлу** — путь к файлу лога; при заполнении лог пишется в файл

```bsl
НастройкиЛога = OPI_NativeAPI.ПолучитьНастройкиЛогирования(Истина, 1000, "C:\Logs\component.log");

// Получение лога
Лог = OPI_NativeAPI.ПолучитьЛог(Хост, Истина, 50); // Последние 50 событий строкой
```

:::tip
Для отладки работы с компонентой рекомендуется включить логирование с записью в файл. Это поможет отследить все вызовы и выявить возможные проблемы
:::

:::important
UUID экземпляра существует только в рамках работы хоста. После уничтожения хоста все созданные экземпляры также будут уничтожены
:::

:::note
Библиотека автоматически преобразует типы данных между 1С и нативной компонентой. Поддерживаются базовые типы: строки, числа, булевы значения, массивы и структуры
:::
