# WebSocket

import LibraryIntro from '@site/src/components/LibraryIntro';

<LibraryIntro module="OPI_WebSocket" cli="ws" use="oint/api/tcp" lang="ru"/>

Этот раздел посвящен библиотеке для работы с WebSocket в 1С:Предприятие, OneScript и CLI. На данной странице описаны все действия, необходимые для полноценного начала работы

## Начало работы

WebSocket — это протокол двусторонней связи, позволяющий устанавливать постоянное соединение между клиентом и сервером. Библиотека предоставляет возможность создания WebSocket сервера для приема входящих соединений.

### Запуск сервера

Для запуска WebSocket сервера необходимо:

1. Определить **порт**, на котором будет работать сервер
2. Указать **размер пула** — максимальное количество одновременно поддерживаемых соединений (по умолчанию 100)
3. При необходимости настроить **логирование** при помощи функции `ПолучитьНастройкиЛогирования`
4. Запустить сервер при помощи функции `ЗапуститьСервер`

```bsl
Порт = 8080;
РазмерПула = 100;

// Настройка логирования (опционально)
НастройкиЛогирования = OPI_WebSocket.ПолучитьНастройкиЛогирования(Истина, 300, "");

ОбъектСервера = OPI_WebSocket.ЗапуститьСервер(Порт, РазмерПула, НастройкиЛогирования);
```

### Настройка логирования

Библиотека поддерживает логирование событий сервера с возможностью записи в память и/или в файл:

```bsl
// Параметры:
// - ЗаписьВПамять: запись лога в память для получения через ПолучитьЛог
// - МаксимумСобытий: максимальное количество событий в памяти
// - ПутьКФайлу: путь к файлу для сохранения полного лога

НастройкиЛогирования = OPI_WebSocket.ПолучитьНастройкиЛогирования(Истина, 300, "/путь/к/логу.log");
```

### Работа с соединениями

После запуска сервер принимает входящие WebSocket соединения. Для работы с данными доступны следующие методы:

**Получение данных из очереди:**

```bsl
// Получает данные из буфера очередного активного соединения
Таймаут = 1000; // мс
МаксимальныйРазмер = 8192; // байт

Результат = OPI_WebSocket.ПолучитьДанныеОчередногоСоединения(ОбъектСервера, Таймаут, МаксимальныйРазмер);
```

**Получение данных конкретного соединения:**

```bsl
// Получает данные из буфера конкретного соединения по ID
IDСоединения = "connection_id_123";

Результат = OPI_WebSocket.ПолучитьДанныеСоединения(ОбъектСервера, IDСоединения, Таймаут, МаксимальныйРазмер);
```

**Отправка данных:**

```bsl
// Отправляет данные в указанное соединение
Данные = "Текстовое сообщение"; // или ДвоичныеДанные

Результат = OPI_WebSocket.ОтправитьДанные(ОбъектСервера, IDСоединения, Данные);
```

**Управление соединениями:**

```bsl
// Получить список активных соединений
СписокСоединений = OPI_WebSocket.ПолучитьСписокСоединений(ОбъектСервера);

// Закрыть конкретное соединение
Результат = OPI_WebSocket.ЗакрытьВходящееСоединение(ОбъектСервера, IDСоединения);
```

:::note
В списке соединений выводятся активные соединения и закрытые соединения с необработанными данными в буфере
:::

### Получение логов

При включенном логировании в память можно получить данные лога:

```bsl
// Получить лог как массив событий
Лог = OPI_WebSocket.ПолучитьЛог(ОбъектСервера, Ложь, 100);

// Получить лог как строку
ЛогСтрокой = OPI_WebSocket.ПолучитьЛог(ОбъектСервера, Истина, 100);
```

### Остановка сервера

Для корректного завершения работы сервера используйте:

```bsl
Результат = OPI_WebSocket.ОстановитьСервер(ОбъектСервера);
```

### Проверка типов объектов

Библиотека предоставляет методы для проверки типов объектов:

```bsl
// Проверка объекта сервера
ЭтоСервер = OPI_WebSocket.ЭтоОбъектСервера(ОбъектСервера);

// Проверка объекта клиента
ЭтоКлиент = OPI_WebSocket.ЭтоОбъектКлиента(ОбъектКлиента);
```
